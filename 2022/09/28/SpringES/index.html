01.Spring Data 简介

what：是什么?

Spring Data是一个用于简化持久层数据访问的开源框架。
其主要目标是使得对数据的访问变得方便快捷。
Spring Data可以极大的简化数据操作的写法，可以在几乎不用写实现的情况下，实现对数据的访问和操作。
包括CRUD外，还包括如分页、排序等一些常用的功能，几乎可以节省持久层代码80%以上的编码工作量。
Spring Data的官网：http://projects.spring.io/spring-data/

02.Spring Data ElasticSearch简介

what：是什么?

Why 为什么使用

  • Templates as a high-level abstraction for storing, searching, sorting documents and building aggregations.
    模板类 ElasticsearchRestTemplate 作为存储、搜索、排序文档和构建聚合的高级抽象
  • Repositories which for example enable the user to express queries by defining interfaces having customized method names
    允许用户通过定义具有自定义方法名的接口来表示查询的存储库
  • Spring Data ElasticSearch 基于 spring data API 简化 elasticSearch操作，将原始操作elasticSearch的客户端API 进行封装 。
  • Spring Data为Elasticsearch项目提供集成搜索引擎。
  • Spring Data Elasticsearch POJO的关键功能区域为中心的模型与Elastichsearch交互文档和轻松地编写一个存储库数据访问层。

官方网站：http://projects.spring.io/spring-data-elasticsearch/

7.ElasticSearch Operations (最重要的模块!!!!!)

IndexOperations定义索引级别的操作，例如创建或删除索引。

DocumentOperations 提供了 新增、更新、删除、getById es文档的操作。

SearchOperations提供了查询搜索es文档的操作

ElasticsearchOperations API结合DocumentOperations和SearchOperations接口。

7.1 创建ElasticsearchRestTemplate implements ElasticsearchOperations

The ElasticsearchRestTemplate is an implementation of the ElasticsearchOperations interface using the High Level REST Client.
ElasticsearchRestTemplate 是ElasticsearchOperations 的实现类。
以下是 使用ElasticsearchOperations 注入到 RestController (可以作为参考注入到 业务层)

7.2 Operations(Query) ,org.springframework.data.elasticsearch.core.query

Query是一个接口，Spring Data Elasticsearch 提供了三种实现CriteriaQuery：StringQuery和NativeSearchQuery
SearchOperations和ReactiveSearchOperations接口的操作方法都带有一个Query参数，该参数决定了要执行的查询条件。

  • CriteriaQuery:条件查询,实例化Criteria使用其方法来进行查询。
    Criteria API

  • StringQuery: 字符串查询

  • NativeSearchQuery:组合查询API

使用原生的的QueryBuidlers生成不同的Query, 传入NativeSearchQuery 进行不同的查询。

7.3 结果集解析 Seach Result Types

SearchHit<T>:Contains the following information:

Id
Score
Sort Values
Highlight fields
Inner hits (this is an embedded SearchHits object containing eventually returned inner hits)
The retrieved entity of type

SearchHits<T>:Contains the following information:

Number of total hits
Total hits relation
Maximum score
A list of SearchHitobjects
Returned aggregations
Returned suggest results

SearchPage<T>:Defines a Spring Data Page that contains a SearchHitselement and can be used for paging access using repository methods.

SearchScrollHits<T>:Returned by the low level scroll API functions in ElasticsearchRestTemplate, it enriches a SearchHitswith the Elasticsearch scroll id.

SearchHitsIterator<T>:An Iterator returned by the streaming functions of the SearchOperations interface.

ReactiveSearchHits:ReactiveSearchOperations has methods returning a Mono<ReactiveSearchHits>, this contains the same information as a SearchHitsobject, but will provide the contained SearchHitobjects as a Flux<SearchHit> and not as a list.

8.2 自定义方法名称查询(重点)

Spring - data - repository 的查询方式

  1. 继承ElasticsearchRepository 使用search()方法查询()

search() 方法

官网注释:继承ElasticsearchRepository的接口 的search() 等方法已被舍弃。

Deprecated since 4.0, use standard repository method naming
or @Query annotated methods
or org.springframework.data.elasticsearch.core.ElasticsearchOperations

Spring-Data-Repository提供了非常强大的自定义查询功能；只要遵循SpringData提供的语法，我们可以任意定义方法声明

官方解释:
Spring Data’s method parsing uses prefix keywords like find, exists, count, and delete and a > terminating By keyword. Everything you put in between find and By makes your method name more
expressive and does not affect query derivation

  • 查询语法：
    基本搜索: find + AnythingYouWantToPutHere + By + [关键词conditions]



多条件搜索: find + AnythingYouWantToPutHere By + 字段名1 + [关键词conditions] + 字段名2

@Query 查询

现在可以使用基于 @Query 在Dao 持久层接口的方法上拼接 DSL 语句,但不推荐,因为编写难度大。
